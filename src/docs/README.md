
# Project Documentation

This directory contains documentation for the Online Store application.

## File Structure

- `components/` - React components organized by category
- `config/` - Application configuration
- `constants/` - Application constants
- `hooks/` - Custom React hooks
- `pages/` - Page components
- `routes/` - Application routing configuration
- `services/` - API services
- `store/` - Redux store configuration
- `styles/` - Global styles and CSS variables
- `types/` - TypeScript type definitions
- `utils/` - Utility functions

## Development Guidelines

1. Follow the existing file structure and naming conventions
2. Use TypeScript for all new code
3. Write components as functional components with hooks
4. Keep components focused on a single responsibility
5. Use proper TypeScript typing for all props and state
6. Write responsive CSS using the defined CSS variables
7. Use the utility functions in the `utils` directory when applicable
8. Follow the established patterns for API calls and state management
